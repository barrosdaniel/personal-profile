Project Idea

Australian Electronic Tourist Guide

Application Overview

The Australian Electronic Tourist Guide is a web-based application designed to assist road travellers to enjoy their Australian road trips to the fullest. It can be accessed on a desktop or tablet, but is more suited to be accessed on a mobile device browser.

It helps users by playing an informational audio when users are on their vehicles and approach certain landmarks along Australian roads. These landmarks can be tourist attractions open to the public or simply a place where important events took place, an illustrious inhabitant that lived in the area, remarkable natural landscapes, important economic activity and much more. The content options have no limits.

Motivation

Have you ever been bored while on the road and looked out the window of your car, bus, or train and wondered about the place you are passing by? So you know exactly what the feeling is like.

Australia is incredibly vast and there is so much to see and do along our roads. There are uncountable stretches of road with jaw-dropping landscapes and interesting histories that attract thousands of national and international tourists each year.

However, the availability of information for visitors using our roads is limited to written media in books, booklets and tourist guides, both published in paper or online. Although information provided in these media assist with information about places and events, they do not make the trips themselves more interesting and fun. The Australian Electronic Tourist Guide explores this opportunity.

Feature Description

The Australian Electronic Tourist Guide is a web-based application that users start up by accessing a URL on a mobile device browser (preferred). The user interface is simple and clean. As it is optimised for small screens (approximately 400 x 600px) not a lot of information is displayed at any one time.

The application website opens up showing a lean navigation bar up the top. The navigation bar has a logo and a hamburger navigation button with the following links: Home, Routes, Login.

Home Page

The Home page has a description of the service with illustrating images and information on the cost of the service and possible plans. The information is presented in text for the information and by tiles with images and descriptions for the levels of service. At the top there is a call-to-action button inviting users to sign up for the service. There is another call-to-action button inviting the user who is not ready to sign up just yet to view the Routes page. A Footer section at the bottom of the page links to the application social media channels and other affiliated tourism service providers and government links.

Routes

The Routes page is accessible by pressing the Home page specific call-to-action button and the navigation button link. Suggested route tiles are listed sorted by region. Similar to the Home page, at the top there is a call-to-action button inviting visitors to sign up for the service.

Sign Up/Login Services

The ‘Sign Up’ feature allows a user to access a shopping cart with basic eCommerce functionality to allow users to subscribe to service plans. This service is linked to credit card providers and charges the user credit/debit card and organises the funds transfer between the parties.

The ‘Login’ feature performs user authentication and checks if a user is authorised to access the ‘Electronic Tourist Guide’ service.

My Routes Services

The ‘My Routes’ feature allows users to search and save routes among the catalogue of routes in the application database. Saved routes are stored in a ‘Favourites’ section of the page. In the ‘Completed Routes’ section, users can indicate which routes they have already completed.

Electronic Tourist Guide Services

The main feature of the application, the ‘Electronic Tourist Guide’ is started when a subscriber user presses the relevant button. It starts a navigation map with the user current location and starts reading the user location in the background. The user location is refreshed every 15 seconds. When a user enters an area of proximity with a landmarked point, an audio file is played on the mobile device by the browser sound player. The proximity parameters are variable according to the user location and the landmark involved to account for differences in landmark density in different areas of the country (e.g. Sydney CBD vs Central NT). The audio player has standard sound controls, such as play/pause, forward/rewind, stop and skip buttons. After the audio stops playing, the app resumes reading the user location and the process starts again.

Tools, Technologies and Skills Required

To implement a tool with the features described above, I believe the technologies below (or variations) would be used as well as resources with knowledge and experience in their implementation and maintenance. Note: by no means this is an exhaustive list and, most likely, as is the case with many web development projects, other tools are added along the way.

  1. VSCode: Visual Studio Code is a lightweight open-source text editor originally created by Microsoft for quickly developing and testing software projects. It has a vast library of plugins to enable and optimise the implementation of all of the technologies described in this summary.
  2. Git is a version control tool that allows the easy tracking and collaboration in software development projects.
  3. Github is a software development project hosting remote repository that works with the Git version control system. It enables collaboration in software development teams.
  4. HTML (Hypertext Markup Language) is a markup language that allows web developers to add content to websites and web applications.
  5. CSS (Cascading Style Sheets) is the style sheet language that allows web developers to add styles and layout to HTML documents.
  6. JavaScript is a scripting language that allows the browser to process interactivity in web sites and web applications.
  7. Geolocation API is a browser-native API that allows web applications to obtain the location from where the user is accessing the application. It powers many applications that use a user’s location, such as web GPS navigators and ridesharing apps.
  8. React is a JavaScript ecosystem that enables quicker and easier web application development by implementing web application design patterns, such as templating and components.
  9. NodeJS is a back-end tool based on JavaScript that is designed to power servers that can run web applications written in JavaScript.
  10. MySQL is a relational database product that can be used to host data for a multitude of application designs.
  11. Progressive Web Apps is a set of tools and principles originally created by Google to implement web applications that look and feel like native mobile applications.
  12. AWS is a cloud-based application hosting service that has the capability to hold the Australian Electronic Tourist Guide application servers and databases with the ability to scale up as needed.
  13. Stripe is a software development toolkit that allows web applications to access financial services, such as receiving credit card payments. It can be implemented as a plugin JavaScript component.
Outcome

The Australian Electronic Tourist Guide will support Local and State governments to provide a better experience to the general public using Australian roads, therefore having an impact on the tourism experience satisfaction. This will, in turn, assist in increasing the profile of the Tourism industry in Australia and to bring more visitors each year into Australia and into the States and territories, generating jobs and business opportunities.
